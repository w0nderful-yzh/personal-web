很多个人网站从选择框架和设计首页开始，最终却停在一组精致的占位卡片上。问题通常不在代码，而在网站还没有明确要长期保存什么。

## 先确定内容之间的关系

这个网站将内容分为三类：

1. **Projects** 负责说明解决过什么问题，以及如何解决。
2. **Blog** 保存已经整理完成、适合完整阅读的文章。
3. **Notes** 保存会持续修改的学习记录。

这三个栏目看起来相近，但更新节奏完全不同。Projects 低频而深入，Blog 中频且结构完整，Notes 高频并允许不完整。

## 首页只是索引

首页的任务不是展示所有内容，而是回答三个问题：

- 这个人是谁？
- 他最近在做什么？
- 从哪里可以进一步了解他？

因此首页只需要精选项目、最近内容和当前动态。完整信息应该留在对应栏目中。

## 选择简单的发布方式

第一版内容使用 Markdown 保存，由 Astro 在构建阶段生成静态页面。发布一篇文章只需要创建文件、预览并提交代码。

```ts
const updateLoop = ['write', 'preview', 'publish'];
```

当这个流程足够轻，网站才有可能真正持续更新。
