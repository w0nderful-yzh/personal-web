# 个人网站内容准备清单

这份清单用于把当前 Demo 替换成真实可上线内容。

## 1. 基础身份信息

- 公开显示名称，例如 `YZH` 或真实姓名
- 一句话介绍，控制在 30-60 个中文字符
- 个人头像或代表性照片
- 所在城市或地区，是否公开
- 邮箱
- GitHub 链接
- LinkedIn、掘金、博客园、知乎等可公开链接
- 当前关注方向，例如后端、前端、系统、AI 应用、工程效率

对应文件：

- `src/data/profile.ts`

## 2. 首页内容

- 首页主标题
- 首页副标题
- 3 个精选项目
- 3-5 条“当前正在做什么”
- 是否需要中英文版本

对应文件：

- `src/pages/index.astro`
- `src/data/profile.ts`
- `src/content/projects`

## 3. 项目材料

每个项目建议准备：

- 项目名称
- 一句话摘要
- 项目时间
- 项目状态：`active`、`completed`、`archived`、`experimental`
- 技术栈
- 你的角色和具体职责
- 项目背景
- 核心功能
- 关键技术决策
- 遇到的问题与解决方案
- 最终结果或指标
- 复盘和下一步
- GitHub 仓库链接
- 在线 Demo 链接
- 2-5 张真实截图或演示图

对应目录：

- `src/content/projects`
- `public/images/projects`

## 4. Blog 技术文章

第一版建议至少准备 1-2 篇完整文章。

每篇文章需要：

- 标题
- 摘要
- 发布时间
- 标签
- 正文
- 代码片段或图片
- 是否公开发布

适合放进 Blog 的内容：

- 完整项目复盘
- 技术方案总结
- 系统性学习文章
- 某个问题的完整分析过程

对应目录：

- `src/content/blog`

## 5. Notes 学习笔记

第一版建议准备 3-5 条短笔记。

每条笔记需要：

- 标题
- 摘要
- 创建时间
- 更新时间
- 所属主题
- 标签
- 状态：`seedling`、`growing`、`evergreen`
- 正文

适合放进 Notes 的内容：

- 命令速记
- 踩坑记录
- 概念摘录
- 学习路线
- 面试题整理
- 课程知识点

对应目录：

- `src/content/notes`

## 6. About 页面

建议准备：

- 个人介绍，200-500 字
- 技术兴趣
- 做项目的方式
- 当前阶段目标
- 可以公开的经历亮点
- 希望别人如何联系你

对应文件：

- `src/pages/about.astro`

## 7. Resume 简历

需要准备：

- 中文或英文简历版本
- PDF 简历
- 教育经历
- 实习或项目经历
- 技能列表
- 奖项、证书或比赛经历
- 联系方式
- 是否需要隐藏手机号、学校、城市等敏感信息

对应文件：

- `src/data/resume.ts`
- `public/resume.pdf`

## 8. 品牌与视觉素材

需要准备：

- 头像
- 网站 favicon
- 首页主视觉是否使用当前生成图
- 项目截图
- Blog 封面是否需要
- 社交分享图风格

对应目录：

- `public/images`

## 9. 上线信息

需要确认：

- 域名是否使用 `yzh666.dev`
- 部署平台：Cloudflare Pages 或 Vercel
- GitHub 主分支名称
- 是否需要访问统计
- 是否需要评论系统
- 是否需要搜索
- 是否公开 RSS

## 10. 发布前必须替换的占位内容

- `src/data/profile.ts` 中的邮箱和简介
- `src/data/resume.ts` 中的教育、经历和技能
- `public/resume.pdf`
- 示例项目内容
- 示例 Blog 和 Notes
- 首页主标题是否保留
- 生成的首页工作台图片是否保留
